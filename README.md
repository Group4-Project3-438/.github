# CST 438 – Project 3 [Name Placeholder]

## Overview
This repository contains our team project for CST 438.  
We are building a full-stack web application with a focus on clean design, testing, and deployment.

This README outlines the **core requirements and direction** for the project during early development.

---

## Key Features (Planned)

### Authentication
- OAuth2 login (Spring Security)  
- At least 2 providers (e.g., Google, GitHub)  
- User account creation, update, and deletion  

### Frontend
- Reactive web application  
- Hosted online (Render, AWS, GCP, etc.)  
- Mobile support (Android or iOS)  
- Consistent UI across platforms  

### Backend
- Spring Boot API  
- At least 8 endpoints  
- Separate deployment from frontend  
- CORS support  

### Database
- PostgreSQL (Supabase recommended)  
- Stores user and app data  

---

## Development Requirements

### Testing
- Frontend: Jest, Playwright  
- Backend: JUnit  
- API: Postman  

### Deployment
- Connected to GitHub  
- Automatic deployments on updates  
- Tests must pass before deployment  

### DevOps
- Backend containerized with Docker  
- CI to run tests on push  

---

## Team Structure
Each team member will build a **full vertical slice**:
- Database (entity/table)  
- Backend (API route)  
- Frontend (UI page)  

---

## MVP Goals
- User authentication (OAuth2)  
- Account creation and management  
- Basic working application flow
